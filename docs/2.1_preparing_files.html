In order to run sample QC and SNP QC, genotype files must be prepared. This takes into account the following:

  1. Unique variants in GigaMUGA array, gm_uwisc_v1.csv, found here (that is, keeping only the probe sequence that appear exactly once in mm10 mouse genome build)
  2. Variants only seen on chromsomes: 1:19, X
  3. Variants seen in DO founder genotypes [which are consensus genotypes for each of the 8 founder lines] and GigaMUGA array
  4. Missing variants across DO founder genotypes & GigaMUGA array

Below is a table representing all variants that were removed at this step.

From here, we follow the steps set out by Karl Broman's to remove further variants before performing quality control checks on samples and snps. More information can be found here. These steps are:

  1. geneseek2qtl2.R

    • see information about preparing data here
    • script is here
    • NB: at this step, GM_processed_files.zip will be used to determine the number of snps to use going forward. These files have consensus genotypes for each founder strain at each SNP, unique SNP alleles, SNP genotypes encoded as A/H/B/-.

  2. geneseek2intensity.R

    • Output is needed to perform array intensity qc, script found here

  3. Calculate genoprobs in order to run sample QC and SNP QC on the raw data through R/qtl2, while also:

    • dropping markers with no data, as mentioned here
    • preparing the files: pr, m and nxo, to perform crossover qc
